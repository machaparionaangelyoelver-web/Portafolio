Fundamentos teóricos

Conceptos clave aplicados en el desarrollo del backend.

Backend y API

El backend procesa solicitudes HTTP, aplica reglas de negocio y devuelve respuestas. En esta práctica, PHP funciona como API para responder en JSON y controlar estados con códigos HTTP.

Concepto de PHP

PHP es un lenguaje del lado servidor. Se ejecuta en el servidor, genera respuestas para el cliente, valida formularios, conecta con bases de datos y permite construir APIs de forma práctica.

PDO y Prepared Statements

PDO facilita la conexión a MySQL con manejo de errores y consultas preparadas. Los prepared statements separan consulta y datos, reduciendo el riesgo de inyección SQL.

Configuración del entorno

Se verificaron versiones de PHP y Composer, se configuró el archivo env, y se organizó el proyecto para mantener rutas, controladores y acceso a datos de forma ordenada.

Objetivos cumplidos

Se completó el flujo real de un backend: recibir la solicitud, validar datos, ejecutar reglas, consultar la base de datos y responder con JSON usando códigos HTTP. Esto permite integrar con frontends (React o Vue) y probar endpoints con Postman.

  • Configurar entorno (XAMPP o Laragon y Composer).
  • Diseñar base de datos relacional en MySQL.
  • Implementar endpoints CRUD.
  • Validar entradas y manejar errores.
  • Probar la API con Postman y registrar evidencias.

Conceptos aplicados

  • CRUD REST: GET, POST, PUT, DELETE con JSON.
  • Flujo: ruta, controlador, base de datos, respuesta.
  • Validación: reglas del servidor y mensajes claros.
  • Seguridad: PDO y prepared statements (anti SQLi).
  • Códigos HTTP: 200, 201, 400, 404, 500 para interpretar resultados.
